jsonp实现跨域的原理

Mar 21, 2016 | Hits

最近的工作是重构所有的注册，之前的注册比较混乱，各个站点各有各的注册，我的工作就是把他们都改为请求新的注册站点。重构的时候遇到了跨域的问题，就直接采用了jsonp实现，之前一直知道jsonp跨域的原理就是利用script标签可以跨域的漏洞，但一直没想过到底是怎么实现的，现在有空好好想一下。

有关浏览器的同源策略，mdn解释的很清楚了，详情看以下几个链接。

jsonp的实现方式

client-side

<script>
    function wakaka(data) {
        console.log(data)
    }
</script>
<script src="http://test.com?callback=wakakka"></script>

server-side

function test(req, res) {
    var callback = req.query.callback 
    var result   = callback + "({ ret : 'wakaka' })"
    res.send(result.ret)
}

上面加载完第一个客户端脚本时，定义了一个函数叫wakaka。

加载完第二个脚本时，向http://test.com发送请求并把wakaka当做callback参数传给服务端。

这时候服务器读取callback参数的值,发现是wakaka，就用wakaka把json数据包裹起来凭借成字符串返回给客户端，也就是wakaka({ ret : ‘wakaka’})这样。

第二个脚本收到服务器的响应，由于是在script标签，就把返回的结果当做是javascript执行，也就是执行 wakaka({ret : ‘wakaka’}),第一个脚本已经定义好了wakaka的函数，执行后就会在终端打印 wakaka。

jquery中jsonp的应用

要在jquery中使用jsonp很简单，如下所示：

$.ajax({
    url : 'http://test.com',

    //传给服务端的回调函数的名字，默认是 callback
    jsonp: "wakaka",

    //告诉jquery我们要使用jsonp
    dataType : "jsonp",  

    success : function(response) {
        //response 就是服务器的返回啦
        console.log(response);
    }  
});

是不是很简单。

虽然是在ajax中使用，但是其实jsonp跟ajax压根不是一回事。

jquery这样做应该是为了我们更方便使用。

我在使用的过程中回调函数名基本上没用，我在服务器的返回也是json数据，也就是上面的 {ret:'wakaka'}, 而不是 wakaka({ret : 'wakaka'})， 然后就直接能在jquery ajax的回调函数success中访问返回结果， 看了看jquery 关于jsonp的实现，没看太明白，改天在写一篇好好研究一下jquery实现jsonp的方式。
